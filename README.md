# 온디바이스 알약 식별 테스트 랩 (웹 프로토타입)

이 저장소는 실제 모바일 MVP에 도입할 온디바이스 파이프라인을 웹 환경에서 검증하기 위한 실험용 도구입니다. 브라우저 카메라로 알약 앞/뒷면을 촬영하면 전처리 → 품질 추정 → 각인 OCR → 로컬 데이터베이스 후보 검색 → 후보 재랭킹 과정을 순차적으로 수행합니다.

## 주요 특징

- **실시간 카메라 캡처**: `getUserMedia` 기반 라이브 뷰에 촬영 가이드를 오버레이하여 실제 촬영 UX를 재현했습니다.
- **이미지 업로드 지원**: 실물 촬영이 어려운 경우 로컬 이미지를 업로드해 동일한 파이프라인을 테스트할 수 있습니다.
- **온디바이스 전처리 & 품질 추정**: 배경제거, 선명도/노출 평가, 분할선/형상 감지, 색상 힌트를 브라우저에서 계산합니다.
- **Tesseract.js OCR 통합**: 각인 텍스트를 인식하고 신뢰도를 추정하여 MFDS 검색 파라미터로 활용합니다.
- **MFDS 데이터 일괄 동기화**: `https://apis.data.go.kr/1471000/MdcinGrnIdntfcInfoService02/getMdcinGrnIdntfcInfoList02` 오퍼레이션과 서비스 키로 전체 낱알 식별 데이터를 내려받아 IndexedDB에 저장합니다.
- **오프라인 후보 검색**: 동기화된 데이터셋을 기반으로 각인/색상/형상 조건을 빠르게 필터링한 뒤 Late-fusion 재랭킹으로 Top-k 후보를 계산합니다.
- **오프라인 스냅샷 자동 대체**: 공공데이터포털 API가 500 오류(Policy Falsified 등)로 응답하면 내장된 샘플 스냅샷을 자동으로 불러와 테스트를 계속 진행할 수 있습니다.
- **후보 재랭킹 로직**: 각인 유사도(Levenshtein), 색상/형상/분할선 일치도, 촬영 품질을 Late-fusion 방식으로 결합합니다.
- **결과 검수 & 재검색**: 추출된 특징을 UI에서 수정하고 곧바로 재검색하여 인식률을 높일 수 있습니다.

> ⚠️ 모든 처리는 로컬 브라우저에서 수행되며, 공공데이터포털 API 호출은 데이터 동기화 시에만 발생합니다.

## 사전 준비

1. [공공데이터포털(data.go.kr)](https://www.data.go.kr/)에서 **식품의약품안전처_의약품 낱알식별 정보** 오픈 API를 신청하고 서비스 키(Encoding/Decoding 버전)를 발급받으면, 자체 프로젝트에서도 안정적으로 사용량을 관리할 수 있습니다.
2. 본 저장소는 안내 이미지와 동일한 엔드포인트(`MdcinGrnIdntfcInfoService02/getMdcinGrnIdntfcInfoList02`)와 테스트용 서비스 키를 UI에 기본 값으로 표시합니다. 별도 입력 없이도 바로 동기화를 수행할 수 있으며, 필요 시 자신의 키로 교체하세요.
3. 데이터 동기화에는 `JSON` 응답이 사용됩니다. `XML`을 선택하면 후보 재랭킹 및 로컬 저장이 동작하지 않습니다.
4. HTTPS 환경(로컬 개발용으로는 `http://localhost` 예외 허용)에서 실행해야 카메라 권한을 받을 수 있습니다.

## 실행 방법

### Node.js 개발 서버

```bash
npm install
npm start
# 기본 포트: http://localhost:4173
```

`npm start`는 경량 Node 서버(`server.js`)를 실행해 정적 파일을 제공하며, `/proxy/mfds` 엔드포인트를 통해 MFDS 오픈 API를 프록시합니다. 브라우저에서 직접 `https://apis.data.go.kr/...`로 호출하면 CORS 정책 때문에 `Failed to fetch` 오류가 발생하므로, 반드시 이 개발 서버(또는 동일한 프록시가 구성된 서버)를 사용해 데이터 동기화를 수행하세요. 만약 정책 변경 등으로 API 호출이 반복해서 실패하면 자동으로 내장된 스냅샷(`data/mfds-snapshot.inline.js` · `data/mfds-snapshot.json`)을 불러와 최소한의 로컬 DB를 구성합니다. 이 스냅샷은 정적 파일 서버 없이 `file://`로 열어도 즉시 적용됩니다.

## 사용 흐름

1. 상단 폼에서 기본으로 표시된 서비스 키를 확인하거나 자신의 키로 교체한 뒤 **전체 동기화** 버튼을 눌러 MFDS 데이터셋을 로컬 IndexedDB에 저장합니다. 동기화 상태는 진행률과 저장 건수로 확인할 수 있습니다. API가 500 오류를 반환하면 화면 하단 로그와 상태 표시가 "오프라인 스냅샷" 대체 여부를 안내합니다. 일부 환경(사파리 비공개 모드 등)에서 IndexedDB가 차단되면 브라우저 메모리로 자동 전환되며, 이 경우 새로고침 시 데이터가 초기화되므로 필요할 때마다 다시 동기화해야 합니다.
2. "카메라 시작"을 눌러 앞/뒷면을 촬영하거나, "이미지 업로드" 버튼으로 미리 준비한 이미지를 불러옵니다. 품질 칩에서 선명도·노출 점수를 확인할 수 있습니다.
3. "파이프라인 실행"을 누르면 각 단계의 진행 상황과 소요 시간이 표시되고, 동기화된 로컬 DB에서 후보가 조회됩니다.
4. 결과 화면에서 Top 후보 카드와 세부 점수를 확인합니다. 필요하면 추출된 각인/색상/형상을 수정한 뒤 "MFDS 재검색" 버튼으로 로컬 DB 재검색을 수행합니다.
5. 하단 텔레메트리 패널과 디버그 로그로 품질 지표와 로컬 검색 시간/진행 상황을 점검합니다.

## 구조 개요

```
├── index.html        # UI 레이아웃 및 접근성 구조
├── styles.css        # 다크 모드 지원 반응형 스타일
└── js/
    ├── app.js        # 상태 관리, 이벤트 핸들링, 파이프라인 오케스트레이션
    ├── camera.js     # 카메라 컨트롤 및 캡처 처리
    ├── image.js      # 전처리, 품질 추정, 형상/분할선 감지 유틸리티
    ├── color.js      # MFDS 색상 매핑 및 팔레트 분석
    ├── ocr.js        # Tesseract.js 기반 각인 OCR 래퍼
    ├── mfds-api.js   # 낱알식별 API 클라이언트 (전체 페이지 페이징 처리 포함)
    ├── database.js   # IndexedDB 캐시 및 로컬 후보 검색 점수화
    ├── pipeline.js   # 단계별 파이프라인 러너
    └── scoring.js    # 후보 재랭킹 및 유사도 계산 로직
├── server.js         # 개발용 정적 파일 서버 (npm start)
├── package.json      # npm 스크립트 및 메타데이터
```

## 한계 및 향후 개선 아이디어

- **모델 정교화**: 현재는 단순 전처리에 의존하며, 실제 모바일 앱에서는 TFLite/ONNX 기반 세그멘테이션·임베딩 모델을 연동해야 합니다.
- **서비스워커/캐시**: 오프라인 업데이트 시나리오를 대비해 모델과 메타 데이터를 캐시하는 PWA 구성이 필요합니다. 현재는 IndexedDB 초기 동기화만 제공합니다.
- **성능 측정**: 다양한 단말에서 FPS, 추론시간, 메모리 사용량을 측정하기 위한 Telemetry 전송 파이프라인을 추가해야 합니다.
- **다국어 OCR**: 각인에 한글/영문/숫자가 혼합되므로 언어 모델 최적화를 통해 인식률을 추가로 높일 수 있습니다.

## 면책

본 도구는 프로토타입으로 의료적 판단이나 실제 복약 안내에 사용할 수 없습니다. 최종 약품 확인은 반드시 의사·약사 등 전문가의 검수를 거쳐야 합니다.
