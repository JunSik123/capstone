알약 식별 테스트 랩

식품의약품안전처(MFDS) 낱알식별 API에서 내려받은 데이터를 로컬에 저장해 온디바이스 식별 파이프라인을 재현한 웹 프로토타입입니다. 실제 모바일 카메라 환경과 비슷한 조건에서 촬영 → 전처리 → 각인 OCR → 로컬 DB 검색 → 후보 재랭킹 과정을 실험해보세요.

1. API 및 장치 준비

공공데이터포털 서비스 키를 등록하고 로컬 데이터베이스를 동기화하세요.

엔드포인트

기본 서비스 키

Encoding
Decoding
키는 브라우저 로컬 저장소에만 보관되며 언제든 삭제할 수 있습니다.

로컬 데이터베이스

동기화 필요

버튼을 눌러 로컬 DB를 준비하세요.

2. 카메라 캡처

앞/뒷면을 동일한 조건으로 촬영하면 인식률이 올라갑니다.
카메라 대기 중

팁: 밝은 배경에서 수직으로 촬영하고 반사를 줄이면 OCR 정확도가 높아집니다.

3. 온디바이스 파이프라인 진행 상황

실제 모바일 파이프라인과 동일한 단계를 시각화합니다.

    4. 추출된 특징 검수 & 보정

    필요 시 각인을 수정하거나 색상을 변경해 재검색할 수 있습니다.

    5. 후보 결과

    동기화된 낱알 식별 데이터와 파이프라인 점수를 결합해 Top 후보를 제시합니다.

    6. 품질 지표 & 로그

    촬영 품질, OCR 확신도, 로컬 검색 시간을 확인하세요.

    테스트 랩 사용 가이드
    
      
      
        
    1. 공공데이터포털(data.go.kr)에서 "식품의약품안전처_의약품 낱알식별 정보" 서비스를 신청하여 서비스 키를 발급받습니다.
      2. 
        
    2. 상단의 "서비스 키 저장" 폼에 키를 입력하고 저장하면 이후 브라우저 로컬에 기억합니다.
      3. 
        
    3. 카메라 권한을 허용한 뒤 앞/뒷면을 촬영합니다. 밝은 배경과 수평 유지가 중요합니다.
      4. 
        
    4. "파이프라인 실행"을 누르면 전처리 → 각인 OCR → 로컬 데이터베이스 검색 → 재랭킹이 순차적으로 수행됩니다.
      5. 
        
    5. 필요하면 각인/색상/형상을 직접 수정하고 "재검색"을 통해 후보를 보정합니다.
      6. 
      
    
      
    모든 처리는 브라우저에서 로컬로 수행하며, 데이터 동기화 시에만 공공데이터포털 API로 연결됩니다.
